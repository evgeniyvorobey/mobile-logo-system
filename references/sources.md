# Sources

Use this file to decide where information should come from and how much authority to give it.

## Priority Order

1. Internal brand truth
2. Platform truth
3. Store truth
4. Icon craft systems
5. HCI and perception research
6. Premium/luxury perception research
7. Market scan
8. Inspiration galleries
9. Legal clearance

Never invert this order without a strong reason.

## 1. Internal Brand Truth

Always inspect the project before inventing new brand logic.

Check for:
- existing design files
- prior logo explorations
- icon assets already in use
- product tone and category constraints
- current UI system
- app store screenshots
- color tokens and typography choices

Examples:
- `.pen` files
- Figma frames
- app icon PNGs
- product docs and strategy docs
- screenshots from the real product

Internal truth defines style fit and redesign tolerance.

Important:
- if the product already has a coherent visual language, the first concept family must respect it
- only introduce a hard stylistic reset as an explicit alternative

## 2. Platform Truth

These sources define real implementation and distribution constraints.

- Apple Human Interface Guidelines: App Icons
  Link: https://developer.apple.com/design/human-interface-guidelines/app-icons
  Use for: iOS icon expectations, recognizability, platform alignment

- Apple App Review Guidelines
  Link: https://developer.apple.com/app-store/review/guidelines/
  Use for: copycat, impersonation, metadata-safe icon/name decisions, and review-safe packaging

- Android Developers: Adaptive icons
  Link: https://developer.android.com/develop/ui/views/launch/icon_design_adaptive
  Use for: foreground/background split, safe zone, shaped mask behavior, monochrome layer

- Android Developers: Images and graphics
  Link: https://developer.android.com/design/ui/mobile/guides/layout-and-content/images-graphics
  Use for: Android asset formats, vector drawables, and launcher/splash context

## 3. Store Truth

These sources matter when icon performance in search, listings, or store conversion is part of the brief.

- Apple App Store: Product Page Optimization
  Link: https://developer.apple.com/app-store/product-page-optimization/
  Use for: testing alternate app icons, screenshots, and previews against real App Store traffic

- Google Play Console Help: Best practices for your store listing
  Link: https://support.google.com/googleplay/android-developer/answer/13393723?hl=en
  Use for: current store listing icon presentation, scale, consistency, and anti-misleading rules

## 4. Icon Craft Systems

- Google Material Icons Guide
  Link: https://developers.google.com/fonts/docs/material_icons
  Use for: simple, minimal, readable icon geometry

Use this tier for shape economy, not for brand personality.

## 5. HCI And Perception Research

- Jylhä & Hamari (2019), "An icon that everyone wants to click"
  Link: https://researchportal.tuni.fi/en/publications/an-icon-that-everyone-wants-to-click-how-perceived-aesthetic-qual/
  Alternate full-text landing: https://trepo.tuni.fi/handle/10024/215199
  Use for: what predicts app icon interaction and successfulness

- VISQUAL (2020), "Development of measurement instrument for visual qualities of graphical user interface elements"
  Link: https://trepo.tuni.fi/bitstream/handle/10024/217453/Jylh_Hamari2020_Article_DevelopmentOfMeasurementInstru.pdf?isAllowed=y&sequence=1
  Use for: balance, symmetry, cohesion, perceived excellence/inferiority

- Trapp et al. (2018), app icon similarity and visual search
  Link: https://link.springer.com/article/10.1186/s41235-018-0133-4
  Use for: crowding, distinctiveness, search efficiency

- Cao et al. (2021), app icon color and border shape
  Link: https://www.sciencedirect.com/science/article/pii/S0169814121000780
  Use for: visual search efficiency, border shape effects

- Shen et al. (2024), "Skeuomorphic or flat?"
  Link: https://doi.org/10.1016/j.displa.2024.102813
  Use for: search-vs-recognition tradeoffs between flat and skeuomorphic icon styles

- Zhang et al. (2024), "Less is Better than More"
  Link: https://doi.org/10.1177/00315125241276240
  Use for: when slight tilt and dynamism may improve vividness in anthropomorphic icon concepts

## 6. Premium / Luxury Perception Research

- Wu (2022), inconspicuous luxury branding
  Link: https://www.frontiersin.org/journals/psychology/articles/10.3389/fpsyg.2022.826890/full
  Use for: quiet luxury, subtle signals, authenticity, craft

- Frontiers (2025), generational perspectives on logo complexity and luxury perception
  Link: https://public-pages-files-2025.frontiersin.org/journals/communication/articles/10.3389/fcomm.2025.1475326/pdf
  Use for: complexity vs luxury tradeoffs

- Kato (2022), perceived material quality from lighting cues
  Link: https://journals.sagepub.com/doi/10.1177/18479790221126791
  Use for: when a premium/material round is justified

Important interpretation rule:
- Mobile app icons cannot absorb luxury complexity the same way packaging or editorial logos can.
- When luxury research suggests complexity helps, prefer internal complexity or material nuance, not noisy outer silhouettes.

## 7. Market Scan

Use market scan to identify:
- category clichés
- oversaturated metaphors
- white-space opportunities
- confusion risk with major competitors

Do not copy forms from competitor icons.
Do use the scan to avoid making something interchangeable.

## 8. Inspiration Galleries

Examples:
- Dribbble
- Behance
- Pinterest
- logo galleries

These are inspiration surfaces only.
Use them late, and never as authoritative evidence.

## 9. Legal Clearance

Only use when the user is close to finalizing a mark or asks about risk.

Examples:
- USPTO: https://tmsearch.uspto.gov/
- EUIPO: https://euipo.europa.eu/

This skill does not replace legal review.

## Practical Rules

- Inspect the local project before live research, then use live research to validate constraints and strengthen the direction.
- If platform rules and inspiration conflict, platform rules win.
- If brand truth and trend moodboards conflict, brand truth wins.
- If a premium effect hurts small-size readability, readability wins.
- If browsing is required for freshness or citations, browse first.
