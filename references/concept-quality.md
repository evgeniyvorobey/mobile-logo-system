# Concept Quality

Use this file when generating, reviewing, or refining directions.

The goal is to produce strong, differentiated options instead of decorative drift.

## What A Strong Direction Must Contain

Each direction should have:
- a named idea
- a clear metaphor
- a silhouette statement
- a reduction rule
- a color logic with assigned roles (primary, secondary, accent) and a monochrome mapping
- a wordmark voice or lockup logic
- a construction method: what grid or proportion system will be used to build this mark
- a reason it fits the current product
- a reason it avoids category cliches
- a main risk
- a longevity note: is this direction structural or trend-dependent?

If one of these is missing, the direction is probably underdeveloped.

## Range Without Noise

For first-round exploration, prefer:
- one evolutionary direction
- one adjacent direction
- one stretch direction when allowed

Do not fake variety by changing only:
- gradients
- color palettes
- minor cutouts
- corner softness
- highlight effects

Those are refinements, not distinct concepts.

## Small-Size Discipline

Before treating a concept as serious, explain:
- what remains visible at `29px`
- what remains visible at `60px`
- which detail is optional and can be removed
- whether the concept still works in one color

If you cannot explain the small-size logic, the concept is not ready.

## Distinctiveness Checks

Ask:
- does this collapse into a category cliche?
- could this be confused with a finance, wellness, crypto, meditation, or kids app by accident?
- is the main shape memorable after a brief glance?
- does the concept still feel like this product rather than a moodboard trend?

## Geometric Construction Discipline

Before presenting any concept, state:
- what construction method was used (circle grid, square module, golden ratio, keyline, freehand-then-rationalized)
- what proportions govern the bounding box
- what optical corrections were applied (overshoot, visual center offset, stroke compensation)
- which critical edges align to the construction grid

If the concept was drawn intuitively, it must be rationalized onto a grid before being presented. Freehand sketches are valid exploration tools but not valid deliverables.

See [geometric-craft.md](geometric-craft.md) for the full construction framework.

## Color Craft Discipline

Before presenting any concept, state:
- which harmony model drives the palette (monochromatic, analogous, complementary, etc.)
- the structural role of each color (primary, secondary, accent)
- the approximate area ratio
- what happens to internal hierarchy when converted to grayscale
- what the flat-fill replacement is for any gradient
- what the monochrome mapping is

If the color was chosen by "it looks good," rationalize it before presenting. Aesthetic intuition is a valid starting point but not a valid justification.

See [color-system.md](color-system.md) for the full color framework.

## Improvement Loop

After presenting concepts:
- identify the best candidate
- identify what is holding it back
- propose 2-3 concrete upgrade moves
- say whether the next round should simplify, sharpen, warm up, premiumize, flatten, or de-risk

Good upgrade moves:
- remove one internal detail that clogs reduction
- tighten the outer silhouette
- shift from descriptive metaphor to symbolic metaphor
- align the geometry with the UI system
- separate the icon logic from the wordmark logic more cleanly
- rationalize freehand curves onto a construction grid
- apply optical corrections (overshoot, visual center, stroke compensation)
- improve negative space: make counterforms deliberate rather than accidental
- reduce gradient dependency: prove the shape works flat first
- test in real context: place on a home screen among competitors

Weak upgrade moves:
- make it cooler
- make it pop
- add more depth
- try more colors
- add a gradient to make it more interesting
- add an effect to make it more premium

## Question Policy

Ask questions only if the answers will change the design path materially.

High-value questions usually concern:
- preserve vs replace existing symbol equity
- premium vs approachable tone
- category white-space vs category familiarity
- store optimization vs brand consistency
- legal or trademark boundaries

If the answer would only affect polish, continue with a labeled assumption instead.

## Craft Gate

No concept set should be presented without:
1. a stated construction method for each direction
2. a stated color rationale for each direction
3. at least one context placement (home screen sketch) for the strongest direction
4. a silhouette isolation rendering (flat single-color) for each direction

If time or tools prevent a full context mockup, a rough sketch placement is acceptable, but silhouette isolation is always possible and always required.
