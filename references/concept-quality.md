# Concept Quality

Use this file when generating, reviewing, or refining directions.

The goal is to produce strong, differentiated options instead of decorative drift.

## What A Strong Direction Must Contain

Each direction should have:
- a named idea
- a clear metaphor
- a silhouette statement
- a reduction rule
- a color logic
- a wordmark voice or lockup logic
- a reason it fits the current product
- a reason it avoids category cliches
- a main risk

If one of these is missing, the direction is probably underdeveloped.

## Range Without Noise

For first-round exploration, prefer:
- one evolutionary direction
- one adjacent direction
- one stretch direction when allowed

Do not fake variety by changing only:
- gradients
- color palettes
- minor cutouts
- corner softness
- highlight effects

Those are refinements, not distinct concepts.

## Small-Size Discipline

Before treating a concept as serious, explain:
- what remains visible at `29px`
- what remains visible at `60px`
- which detail is optional and can be removed
- whether the concept still works in one color

If you cannot explain the small-size logic, the concept is not ready.

## Distinctiveness Checks

Ask:
- does this collapse into a category cliche?
- could this be confused with a finance, wellness, crypto, meditation, or kids app by accident?
- is the main shape memorable after a brief glance?
- does the concept still feel like this product rather than a moodboard trend?

## Improvement Loop

After presenting concepts:
- identify the best candidate
- identify what is holding it back
- propose 2-3 concrete upgrade moves
- say whether the next round should simplify, sharpen, warm up, premiumize, flatten, or de-risk

Good upgrade moves:
- remove one internal detail that clogs reduction
- tighten the outer silhouette
- shift from descriptive metaphor to symbolic metaphor
- align the geometry with the UI system
- separate the icon logic from the wordmark logic more cleanly

Weak upgrade moves:
- make it cooler
- make it pop
- add more depth
- try more colors

## Question Policy

Ask questions only if the answers will change the design path materially.

High-value questions usually concern:
- preserve vs replace existing symbol equity
- premium vs approachable tone
- category white-space vs category familiarity
- store optimization vs brand consistency
- legal or trademark boundaries

If the answer would only affect polish, continue with a labeled assumption instead.
