# Example Requests

Use these to recognize when the skill should trigger and what “good” looks like.

## Example 1

User request:
- "Create a logo and app icon for a premium sleep app for parents."

Expected behavior:
- research platform rules
- define 3-5 territories
- generate concepts
- compare them
- refine one winner
- produce iOS + Android + monochrome package notes

## Example 2

User request:
- "We already have a symbol. Make it work as an Android adaptive icon and a flat app icon."

Expected behavior:
- skip big territory exploration
- audit current mark
- run flat and monochrome rounds
- define foreground/background/monochrome logic

## Example 3

User request:
- "This current logo feels generic. Show me stronger mobile-first directions."

Expected behavior:
- identify clichés
- explain why the current mark fails
- generate 3-5 differentiated directions
- score distinctiveness and small-size resilience

## Example 4

User request:
- "Prepare the final asset checklist for handoff."

Expected behavior:
- focus on package-spec
- list missing validations
- produce a concise shipping checklist
