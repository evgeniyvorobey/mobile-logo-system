# Example Requests

Use these to recognize when the skill should trigger and what “good” looks like.

## Example 1: Full Hi-End Workflow

User request:
- “Create a logo and app icon for a premium sleep app for adults. Research platform guidance, define territories with construction grids and color rationale, score with the full matrix, run the craft pass, and context test on a real home screen.”

Expected behavior:
- research current Apple and Android icon guidance
- define 3-5 territories, each with stated construction method, color roles, silhouette logic
- generate concepts built on a grid, not freehand
- score using all 11 dimensions including geometric precision, color craft, context survival, longevity
- recommend the strongest direction and wait for user selection before craft pass
- after selection, run the craft pass: optical corrections, negative space audit, tangent continuity, color accessibility
- context test: iOS home screen (light/dark), App Store listing, competitor row
- define wordmark with justified pairing and measured lockups
- define Android adaptive split and monochrome layer
- produce iOS + Android + monochrome package notes

## Example 2: Existing Symbol Adaptation

User request:
- “We already have a symbol. Make it work as an Android adaptive icon and a flat app icon.”

Expected behavior:
- skip big territory exploration
- audit current mark: extract construction logic, check geometric precision, verify optical corrections
- run flat and monochrome rounds
- define foreground/background/monochrome logic
- test under multiple Android mask shapes and Material You tinting
- verify pixel alignment at 48dp and 29px
- context test: Android home screen with circular and squircle masks

## Example 3: Competitive Differentiation

User request:
- “This current logo feels generic. Show me stronger mobile-first directions that stand out in the category.”

Expected behavior:
- audit current mark: identify category cliches used, silhouette uniqueness failure, color conformity
- map the cliche landscape: what shapes, colors, metaphors are overused in the category
- generate 3-5 differentiated directions, each with construction method, color rationale, and longevity note
- score distinctiveness, small-size resilience, context survival, and longevity
- run competitor row test: place each direction among 5-7 real category icons
- recommend strongest direction and ask the user to choose before craft pass
- run style-agnostic silhouette test (flat, outline, cut-out)

## Example 4: Production Audit

User request:
- “Audit what we have and tell me if this logo system is actually ready to ship.”

Expected behavior:
- evaluate reduction at all target sizes (1024px down to 29px)
- evaluate geometric craft: grid, optical corrections, tangent continuity, path cleanliness
- evaluate color: contrast ratios, grayscale conversion, CVD simulation
- evaluate monochrome layer
- run context test: home screen, store listing, competitor row
- run longevity assessment: style-agnostic silhouette test, 5-year durability question
- check wordmark pairing and kerning
- check package completeness: list every deliverable as pass / revise / missing
- output prioritized punch list ordered by shipping risk

## Example 5: Brand Refresh With Equity Preservation

User request:
- “Our app has millions of users. The icon looks dated but users must still recognize it after the update. Propose evolutionary refinements.”

Expected behavior:
- audit current mark: separate structural identity from stylistic treatment
- render current icon as flat single-color fill to identify the equity core
- propose 2-3 evolutionary refinements that preserve the recognizable silhouette
- add a specific “recognizability preservation” criterion to the scoring matrix
- recommend the strongest refinement and wait for user selection before craft pass
- after selection, run craft pass: rationalize freehand curves onto a grid, improve reduction, modernize without trend-chasing
- context test: show old and new icons side by side on a home screen
- define monochrome path
- prepare handoff package with before/after comparison

## Example 6: Android-First With Monochrome Priority

User request:
- “We are shipping an Android-first utility app. The themed icon must be the primary deliverable. Design monochrome first.”

Expected behavior:
- design the monochrome silhouette as the foundation
- build the full-color version from the monochrome, not the other way around
- test under multiple Material You palettes (warm, cool, neutral, vibrant)
- define foreground/background split with safe-zone discipline
- test all mask shapes (circle, squircle, rounded square, teardrop)
- verify parallax shift tolerance
- context test: Android launcher, themed icon row, notification bar
- adapt for iOS as secondary target
